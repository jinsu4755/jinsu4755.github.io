웹프로그래밍 개념

강의 시간에 배운 내용을 정리합니다.

웹 프로그래밍

클라이언트(Client) 와 서버(Server) 방식으로 아래 사진과 같은 구조를 가진다.

Web Browser

: HTML 문서 내용을 시작적으로 보여주기위한 SW, 인터넷을 통해 웹서버로 부터 HTML 문서를 받아서 보여준다

Web Server

: 요청(Request)이 요구사는 HTML문서를 Web Browser에게 인터넷을 통해서 전달.

전달 받은 HTML문서를 시작적으로 그려서 보여준다.

Request : 주소 입력 or 클릭 시 해당 링크를 보내달라는 요청을 보낸다.

Response : Request 발생 시점 부터 요청에 대한 응답으로 HTML문서를 전달한다.

정적인 / 동적인 문서

정적인 문서(Static Document)

HTML 내용을 그대로 화면에 송출

동적인 문서(Dynamic Document)

사용자의 인터랙션에 따라 웹페이지가 바뀜, JSP처럼 내용이 수정되어서 그려지는 문서를 말함

웹서버 프로그래밍

웹 서버 프로그램(Web Server Program)

파일에 저장된 내용을 바탕으로 웹서버가 무엇인가 실행해 문서 내용을 수정하고 웹 브라우저에 전달할 때, 문서 수정을 위해 웹 서버에 실행되는 소스 코드를 말한다.

JSP(Java Server page)

자바 웹프로그래밍 기술, 내부 Java 코드로 실행하고 변경 내용을 전달한다.
