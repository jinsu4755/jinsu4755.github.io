주제

Python + Git

진행

Python의 개념을 설명하는 것과 동시에 서브로 Git의 사용법을 익힙니다.

Python의 경우 C언어와 차이점에 중점을 두고 꼭 알아두어야할 개념을 위주로 설명합니다.

문법의 변화, 입력이나 변수의 사용시 주의점

list를 다루는 방법, 문자열 슬라이싱, 정규식, class의 개념을 아주 살짝 건드려보는 등

Python뿐 아니라 추후 멘토링을 하는 친구들이 Java의 개념 이전에 객체지향 맛보기 처럼 진행할 예정입니다.

Git의 경우는 제 블로그보다 더 잘 설명되어있는 사이트를 이용합니다.

관련 자료의 경우는 구글에서 더 전문적이고 정확한 지식을 얻을 수 있기 때문입니다.

*깃, 깃허브 아주 간단히 설명하기 *

Git - 소스코드를 버전별로 다른사람과 함께 관리가 가능하다(분산 버전 관리 툴)

Github - Git을 사용하는 프로젝트를 지원하는 웹호스팅 서버, 오픈소스로 자신의 코드를 공유할 시 서버를 무료로 제공해준다.

즉 자신이 만든 프로그램, 혹은 프로젝트를 다른사람과 함께 관리하고 협업할 수 있으며 오픈소스로 코드를 공유할 수도 있다!

Git은 프로그래머로서, 앞으로 프로그래머가 되기위해 꼭 알아야합니다!

왜? 실무에서 필요하기 때문!

Git을 왜 써야하지?

  1. 협업을 더 효과적으로 할 수 있다.

  2. 손쉬운 개발 및 테스트 환경을 구축!

  3. 효율적으로 배포를 관리할 수 있다.

만약 2이상의 사람이 모여 웹서비스를 만든다고 가정하면, 서로 파일을 하루는 누가 하루는 누가 만들고 주고 받으면 비효율적이다.

1이 2에게 파일을 주고 2는 받은파일을 작업하여 서버에 올렸다 치자.

1은 2에게 주었던 파일을 새로 작업하면 충돌이 발생한다.

-> 이미 올라간 코드와, 새로 만든 코드간에 충돌

또, 서비스에 복잡도에 따라서 배포까지 몇번의 과정이 필요하다고 하는데..

GIt을 사용함으로 코드간의 충돌을 막을 수 있고(바로 덮어씌워지지 않음) 언제든 새로운 내용을 기록하여 배포할 수 있으며 문제가 발생해도 문제발생 이전 시점으로 돌아가는 것이 가능하다^^

Kia이거완전 갓갓갓프로그램이자놔?

Git의 장점

학습자의 경우

고급자의 코드를 보고 더 좋은 프로그래밍 방법을 익히고, 고급자의 장점을 보고 따라하면서 습득이 가능하다.

별이 많은 오픈소스 프로젝트를 볼 수 있다.

후에 더 발전하면

오픈소스에 개발자로 참여하거나 의견지원으로 더 나은 프로그램에 기여가 가능하다.

학생 입장에서 사실 Git을 쓰는이유

강의를 듣고 과제, 공부한 내용을 올리고 다른 사람의 공부내용과 코드를 보며 자신의 코드를 더 좋게 바꾸어 갈 수있다.

공부한 내용과 과제를 올리며 일종의 포트폴리오로 사용하면서 자신이 얼마나 열심히 공부하고 열심히 코딩하는지 보여줄 수 있다.
